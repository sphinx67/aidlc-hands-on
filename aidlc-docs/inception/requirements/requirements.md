# 테이블오더 서비스 요구사항 문서

## 인텐트 분석 요약

- **사용자 요청**: 테이블 오더 앱 신규 개발 (requirements 폴더 기반)
- **요청 유형**: New Project (Greenfield)
- **범위 추정**: System-wide (고객용 UI + 관리자용 UI + 백엔드 API + DB)
- **복잡도 추정**: Complex (다수 컴포넌트, 실시간 SSE 통신, 세션 관리, JWT 인증)

---

## 기술 스택 결정사항

| 항목 | 선택 |
|------|------|
| 백엔드 | Python + FastAPI |
| 프론트엔드 | React (TypeScript) |
| 데이터베이스 | SQLite |
| 배포 방식 | 백엔드 API 서버 + 프론트엔드 정적 파일 분리 배포 |
| 이미지 처리 | 이미지 URL만 저장 (업로드 없음) |
| 화면 구성 | 단일 웹앱 내 라우팅으로 구분 (/customer, /admin) |
| 초기 데이터 | 시드 데이터 스크립트로 자동 생성 |
| 메뉴 관리 | MVP 범위에서 제외 |

## 1. 기능 요구사항 (Functional Requirements)

### 1.1 고객용 기능

#### FR-C-01: 테이블 자동 로그인 및 세션 관리
- 관리자가 1회 설정: 매장 식별자, 테이블 번호, 테이블 비밀번호 입력
- 설정 정보를 브라우저 localStorage에 저장
- 이후 앱 접속 시 저장된 정보로 자동 로그인
- 로그인 성공 시 세션 ID 발급 및 저장

#### FR-C-02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본(홈) 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 카드: 메뉴명, 가격, 설명, 이미지(URL) 표시
- 카테고리 간 빠른 이동 (탭 또는 사이드바)
- 카드 형태 레이아웃, 터치 친화적 버튼 (최소 44x44px)

#### FR-C-03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절 (증가/감소)
- 총 금액 실시간 계산
- 장바구니 비우기
- localStorage에 저장 (페이지 새로고침 시 유지)
- 서버 전송은 주문 확정 시에만 수행

#### FR-C-04: 주문 생성
- 주문 내역 최종 확인 화면
- 주문 확정 버튼
- 주문 성공 시: 주문 번호 5초 표시 → 장바구니 비우기 → 메뉴 화면 자동 리다이렉트
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 식별자, 테이블 식별자, 세션 ID, 메뉴 목록(메뉴명/수량/단가), 총 금액

#### FR-C-05: 주문 내역 조회
- 현재 테이블 세션의 주문만 표시 (이전 세션 제외)
- 주문 시간 순 정렬
- 주문별: 주문 번호, 주문 시각, 메뉴 및 수량, 금액, 상태(대기중/준비중/완료)
- 페이지네이션 또는 무한 스크롤

### 1.2 관리자용 기능

#### FR-A-01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호로 로그인
- JWT 토큰 기반 인증, 16시간 세션 유지
- 브라우저 새로고침 시 세션 유지
- 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱 저장

#### FR-A-02: 실시간 주문 모니터링
- Server-Sent Events (SSE) 기반 실시간 주문 업데이트
- 2초 이내 신규 주문 표시
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 n개 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경: 대기중 → 준비중 → 완료
- 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- 테이블별 필터링

#### FR-A-03: 테이블 관리
- **테이블 초기 설정**: 테이블 번호, 비밀번호 설정, 16시간 세션 생성
- **주문 삭제**: 특정 주문 삭제 (확인 팝업 → 즉시 삭제 → 총 주문액 재계산)
- **테이블 세션 종료(이용 완료)**:
  - 확인 팝업 표시
  - 현재 세션 주문 내역을 OrderHistory 테이블로 이동
  - 테이블 현재 주문 목록 및 총 주문액 0으로 리셋
  - 새 고객이 이전 주문 내역 없이 시작 가능
- **과거 주문 내역 조회**:
  - 테이블별 과거 주문 목록 (시간 역순)
  - 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각
  - 날짜 필터링

## 2. 비기능 요구사항 (Non-Functional Requirements)

### 2.1 성능
- SSE 기반 실시간 업데이트: 신규 주문 2초 이내 표시
- 메뉴 목록 로딩: 3초 이내

### 2.2 보안
- JWT 토큰 기반 관리자 인증 (16시간 만료)
- 비밀번호 bcrypt 해싱
- 테이블 인증: 매장 식별자 + 테이블 번호 + 비밀번호 조합

### 2.3 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 카드 형태의 직관적 레이아웃
- 명확한 시각적 계층 구조

### 2.4 데이터 지속성
- 장바구니: localStorage (새로고침 유지)
- 테이블 로그인 정보: localStorage (자동 로그인)
- 주문 이력: SQLite DB 영구 저장

## 4. MVP 개발 범위

### 고객용
- FR-C-01: 테이블 자동 로그인 및 세션 관리
- FR-C-02: 메뉴 조회 및 탐색
- FR-C-03: 장바구니 관리
- FR-C-04: 주문 생성 (5초 표시 후 자동 리다이렉트)
- FR-C-05: 주문 내역 조회 (현재 세션만)

### 관리자용
- FR-A-01: 매장 인증 (16시간 JWT 세션)
- FR-A-02: 실시간 주문 모니터링 (그리드 레이아웃, SSE)
- FR-A-03: 테이블 관리 (초기 설정, 주문 삭제, 이용 완료, 과거 내역 조회)

## 5. 데이터 모델 개요

### 주요 엔티티
- **Store**: 매장 정보 (식별자, 이름, 관리자 계정)
- **Table**: 테이블 정보 (번호, 비밀번호, 현재 세션 ID)
- **Category**: 메뉴 카테고리
- **Menu**: 메뉴 정보 (이름, 가격, 설명, 이미지 URL, 카테고리)
- **Order**: 주문 정보 (테이블, 세션 ID, 상태, 총 금액, 시각)
- **OrderItem**: 주문 항목 (주문, 메뉴, 수량, 단가)
- **OrderHistory**: 과거 주문 이력 (세션 종료 시 이동)
